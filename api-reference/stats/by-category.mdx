---
title: "Get expenses by category"
description: "Returns total expenses grouped by category for a given time period."
---

Returns an array of expense totals grouped by category, optionally filtered by month and/or year. Only expense-type transactions are included.

```bash
GET /api/stats/by-category
```

## Request

### Query parameters

<ParamField query="month" type="number">
  Month to filter by. Must be an integer between 1 and 12.
</ParamField>

<ParamField query="year" type="number">
  Year to filter by. Must be within the range of years present in your transaction data.
</ParamField>

## Response

Returns an array of category totals. Returns an empty array (`[]`) if there are no expense transactions in the specified period.

<ResponseField name="name" type="string">
  Category name, capitalized (e.g., `"Food"`, `"Transport"`).
</ResponseField>

<ResponseField name="value" type="number">
  Total amount spent in this category.
</ResponseField>

<Info>Income transactions are excluded. Only expense transactions contribute to the totals.</Info>

## Errors

| Status | Error message | Cause |
|--------|---------------|-------|
| 400 | `The month must be a number` | `month` query param is not a valid number |
| 404 | `Invalid month. Range: 1–12` | `month` is a number but outside the 1–12 range |
| 400 | `Invalid year. Available range: <min>–<max>` | `year` is outside the range of years in your data |

## Example

```bash
curl http://localhost:3000/api/stats/by-category?month=3&year=2024
```

```json
[
  { "name": "Food", "value": 420 },
  { "name": "Transport", "value": 210 },
  { "name": "Entertainment", "value": 150 },
  { "name": "Health", "value": 185 }
]
```
